> 9 步上手，从第一句对话到全自动运行。

## 一、搭建环境 — 只需做一次

### 1. 创建并置顶持久线程

**第一次打开 Codex 时**

📌 新建线程 → 输入名称（如「日常助手」）→ 点击线程标题旁的 **Pin** 按钮置顶

⌨ 之后用 `Cmd+1` ~ `Cmd+9` 直接跳到对应置顶线程

**建议置顶的线程类型：**
- Chief of Staff
- 发布线程
- 文档审阅
- 外部监控

**效果：** Codex 在这个线程里保留你所有的决策、偏好和上下文，下次打开不用从头解释。

---

### 2. 设置共享记忆 & AGENTS.md

**置顶线程之后**

📁 本地建文件夹 `~/vault`，结构如下：

```
vault/
├── AGENTS.md
├── TODO.md
├── people/
├── projects/
└── notes/
```

**在 AGENTS.md 里写：**

> 把 `~/vault` 当作持久工作记忆。记录：人员、项目、决策、待办、阻塞项。优先维护已有笔记，不要制造笔记膨胀。没有实质变化时不要改动 vault。

**同时开启：**

⚙ Settings → Personalization → Memories，开启 Codex 内置记忆（偏好、常用流程、已知坑）。再开启 **Chronicle** 让 Codex 从屏幕上下文自动学习。

**效果：** 重要信息写进文件夹 + Codex 自带记忆双保险，换线程、换设备都能接上。

---

## 二、发起任务 — 每次工作开始时

### 3. 用语音输入说出想法

**有模糊想法、懒得打字时**

🎤 点击 Codex 输入框旁的**麦克风图标**，直接说话（支持 2-3 分钟连续口述）

**示例：**

> 我记得 Ben 在 Slack 里提过一个关于登录页的事，具体记不清了，帮我去翻一下相关讨论。

也可以用 📋 粘贴一段会议转录或口述笔记作为输入，比摘要保留更多细节。

**效果：** Codex 拿到原始思路后自动搜索 Slack、Gmail、Calendar，整理结果汇报给你。

---

### 4. 用 `$browser` / `@chrome` / `@computer` 接上工具

**任务涉及代码以外的操作时**

在输入框里用 `@` 或 `$` 调用：

| 指令 | 功能 |
|------|------|
| `$browser` | 在 Codex 侧边栏内打开网页，Codex 可以审查、标注、操控页面 |
| `@chrome` | 复用你 Chrome 已登录的状态，操作需要登录的网站 |
| `@computer` | 操控桌面 GUI 应用（只通过界面才能完成的工作） |

**MCP 连接器：** Slack / Gmail / Calendar

**示例：**

> 用 `$browser` 打开 staging.example.com，截图审查首页样式。

🔁 **可复用的 Skills：** 把常用工作流打包成 Skill，之后 Codex 可以直接重跑，不用每次重新教。

**效果：** Codex 从"只能写代码"变成"能操作整个电脑和外部服务"。

## 三、在线协作 — 你在屏幕前时

### 5. Steering 打断 & Queuing 排队

**Codex 正在执行任务时**

**Steering** — 在侧边栏标注或在输入框打字，打断当前动作

✋ Codex 正在审网页时，直接在侧边栏的页面上点击标注，或在底部输入框输入修正：

> make this smaller / 这两个元素间距太大了 / 这段文案不对，改成…

**Queuing** — 在输入框里追加下一条指令，不打断当前任务

📋 当前任务继续跑，你输入的内容排到队列，等当前步骤完成后自动执行：

> 等这步做完，把预览链接发到 Slack #design 频道。

**效果：** Steering = 当场转弯 · Queuing = 设下一个目的地。你始终掌控节奏，不用等 Codex 跑完才能介入。

---

### 6. 在侧边栏审查 & 标注产出

**Codex 产出代码、网页、文档后**

👁 侧边栏自动弹出产出物 → 直接在上面点击、标注 → Codex 读取标注后立刻修改

**侧边栏支持审查的内容：**
- 代码 diff / Markdown / 数据表格 / PDF / 幻灯片 / 网页（`$browser`）

**特别好用的审查场景：**
- `index.html` 静态原型
- Storybook 组件
- Remotion 动画
- 数据分析 App

**效果：** 不用导出、不用切窗口。标注即修改，审查和修正在同一个循环里闭环。

## 四、离线自动 — 你离开座位时

### 7. 用 Automations 设置定时 / 线程自动任务

**有重复性工作时**

**Scheduled Automation** — 每次全新开始

⏰ 进入 **Automations** 面板 → 新建 → 选 Schedule → 设时间（如每天 9:00）

> 每天早上 9 点检查仓库 CI 状态，失败的汇总成报告。

**Thread Automation** — 定期唤醒同一线程，延续上下文

💓 在置顶线程里进入 **Automations** → 选 Thread Automation → 设间隔（如每 30 分钟）

> 每 30 分钟检查 Slack 和 Gmail，筛选需要我回复的消息，调研背景并草拟回复，但不要发送。

**线程自动化还能做的：**
- 监听 PR 评论 / Google Docs 评论 / Slack 回复 → 自动跟进

**效果：** 你回来时，信息已收集好、回复已草拟好。你只负责审核和发送。

---

### 8. 用 Goals 设定可验证的完成标准

**复杂任务、需要持续执行时**

❌ 差的指令（没有终点）：

> 把这个模块用 Rust 重写一下。

✅ 好的指令（Goal + Verifier）：

> 用 Rust 重写 `src/auth` 模块。完成标准：全部现有单元测试通过。如果测试失败，继续修直到通过为止。

**可用的 Verifier 类型：**
- 测试套件 / 性能基准 / Bug 复现用例 / 验证矩阵 / E2E 流程

**效果：** Codex 持续推进直到验证器通过。有明确终点，而不是无限改下去。

---

## 五、移动跟进 — 你在路上时

### 9. 用 Codex 移动端查看进度、拍板决策

**离开电脑但 Codex 还在跑时**

📱 打开 **Codex 移动端 App** → 进入线程 → 查看产出 → 回复 / 批准 / 用语音重新引导

**示例：**

> 方案 A 就行，继续往下走。
> 等等，先看一下方案 B 的测试结果再决定。

💻 **关键点：** Mac 上的文件、权限、本地环境全部保留不动 — 手机只是远程遥控器。

**效果：** 你的决定即时生效。回到电脑前时，进度已经推进了。
